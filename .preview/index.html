Unified Approvals Connector (1.0)

Download OpenAPI specification:

The Unified Approvals Connector is a Fastify service that bridges Workfront (Redrock) and the Unified Approvals system. It manages document-version approvals, multi-stage workflows, participant management, template CRUD, and decision-making.

Authentication

The connector supports one authentication method:

Method Header/Cookie Use case
Bearer Authorization: Bearer <token> IMS Bearer token from Adobe IO / Developer Console

All endpoints accept Bearer token auth.

For Bearer token auth, the following Workfront context headers are also required:

  • x-api-key — API key from Adobe Developer Console
  • x-gw-subdomain — customer Workfront subdomain
  • wf-customerid — Workfront customer ID
  • wf-userid — Workfront user ID
  • wf-lane — Workfront deployment lane (e.g. prod, devtest)

Asset Types

Endpoints currently only support DOCV (Workfront document version) as the assetType path parameter.
